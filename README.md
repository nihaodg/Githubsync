# GitHub Sync 使用教程

## 一、配置 Token

### 1.1 生成 GitHub Token

1. 打开 GitHub：https://github.com
2. 点击右上角头像 → **Settings**
3. 左侧菜单找到 **Developer settings**
4. 点击 **Personal access tokens** → **Tokens (classic)**
5. 点击 **Generate new token (classic)**
6. 勾选权限：**repo**（完全控制私有仓库）
7. 点击 **Generate token**
8. **立即复制 Token**（只显示一次！）

### 1.2 在软件中配置

1. 打开 `GitHubSync.exe`
2. 在 Token 输入框粘贴你的 GitHub Token
3. Storage Path 可以留空（使用默认路径）
4. 点击 **保存并连接**

---

## 二、克隆仓库

### 2.1 首次使用需要先克隆

1. 在软件主界面找到 **克隆仓库** 输入框
2. 输入要克隆的 GitHub 仓库地址，例如：
   ```
   https://github.com/nihaodg/Githubsync
   ```
3. 本地文件夹名称可以留空，会自动使用仓库名
4. 点击 **克隆** 按钮
5. 等待克隆完成（首次可能需要一些时间）

### 2.2 仓库存储位置

- **默认路径**：`C:\Users\你的用户名\GitHubSync\repos`
- **自定义路径**：在 Storage Path 中填写完整路径

---

## 三、日常使用：拉取和推送

### 3.1 拉取（Pull）- 获取远程最新代码

**使用场景**：你在另一台电脑或网页上修改了代码，想同步到当前电脑

**操作步骤**：
1. 在左侧 **仓库列表** 中点击选择要更新的仓库
2. 点击右侧的 **拉取** 按钮
3. 等待拉取完成
4. 查看文件状态确认是否同步

### 3.2 推送（Push）- 上传本地提交到远程

**使用场景**：你在本地修改了代码，想要同步到 GitHub

**操作步骤**：
1. 修改本地文件（可以用任何编辑器）
2. 在软件中点击选择仓库
3. 查看 **文件状态**，确认有修改的文件
4. 在 **提交更改** 输入框中填写提交说明，例如：
   ```
   修复了登录页面的显示问题
   ```
5. 点击 **提交** 按钮
6. 点击 **推送** 按钮
7. 等待推送完成

## 四、常见问题

### Q: 克隆失败怎么办？
- 检查 Token 是否正确
- 检查网络连接
- 确认仓库地址拼写正确

### Q: 推送失败怎么办？
- 确认网络正常
- 确认 Token 有 repo 权限
- 先尝试 **拉取** 再推送（可能远程有新代码）

### Q: 提交和推送有什么区别？
- **提交 (Commit)**：把修改保存到本地记录
- **推送 (Push)**：把本地提交上传到 GitHub
- 必须先提交才能推送

### Q: 如何确认同步成功？
- 推送后去 GitHub 仓库页面刷新
- 查看提交记录是否显示最新的提交

## 五、完整操作流程示例

### 场景：你在公司修改了代码，回家后继续开发

```
公司电脑操作：
1. 拉取最新代码（确保没有遗漏）
2. 修改文件
3. 提交并推送

回家后电脑操作：
1. 拉取最新代码（获取公司的修改）
2. 继续修改
3. 提交并推送
```

### 场景：网页上直接修改了代码

```
网页操作：
1. 在 github.com 上直接修改文件
2. 提交更改

本地软件操作：
1. 选择仓库
2. 点击拉取（获取网页上的修改）
```

---

## 六、注意事项

1. **Token 安全**：不要在不信任的电脑上保存 Token
2. **定期同步**：经常拉取可以避免合并冲突
3. **提交说明**：养成写清楚提交说明的习惯
4. **网络要求**：拉取和推送都需要网络连接
